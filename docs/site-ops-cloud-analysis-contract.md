# Site Ops Cloud Analysis Contract

Status: contract-first handoff for a future Cloud implementation.

Operations Insights remains a Toolbox local operator surface. The local plugin
collects bounded public WordPress evidence and builds
`site_ops_insight_pack.v1`. For heavier analysis, Toolbox may prepare
`site_ops_cloud_analysis_request.v1`, but it must not call Cloud until a
separate Cloud runtime implementation and host integration are accepted.

## Request Contract

`site_ops_cloud_analysis_request.v1` is a privacy-minimized runtime/detail
request. It contains:

- site name, description, and host;
- aggregate post/page, media, comment, and taxonomy sample summaries;
- ranked local findings from `site_ops_insight_pack.v1`;
- blocked items and requested analysis tasks;
- the expected result contract `site_ops_cloud_analysis_result.v1`.

The request may ask Cloud to:

- prioritize the operator review queue;
- explain metric drivers;
- identify repeated public comment needs from counts and signals;
- rank content refresh candidates;
- rank media metadata review candidates;
- prepare Core handoff candidates without creating proposals.

## Cloud Boundary

Cloud role is `runtime_detail`. Cloud may return deeper ranking, trend notes,
confidence, evidence refs, blocked items, Core handoff candidates, and operator
next actions. Cloud must not become a second ability registry, workflow
registry, approval store, scheduler truth, or WordPress write owner.

The request contract explicitly keeps:

- `write_posture=suggestion_only`;
- `direct_wordpress_write=false`;
- `core_proposal_created=false`;
- `local_runtime_created=false`;
- `local_scheduler_created=false`;
- `cloud_called=false` when generated by local preview.

## Privacy Boundary

The request must not include full comment text, comment author email, comment IP
address, user agent, provider secrets, raw logs, or private WordPress content.
Approved public comments are represented only as aggregate signal counts.

## Future Implementation Gate

Before enabling a real Cloud call, implement and verify the Cloud-side
`site_ops_cloud_analysis_result.v1` response in `magick-ai-cloud`. Toolbox may
then add a narrow host/runtime bridge that submits this request and displays the
returned suggestion-only result. Do not add local queues, custom run tables,
Action Scheduler, automatic proposal creation, or WordPress writes in Toolbox.
